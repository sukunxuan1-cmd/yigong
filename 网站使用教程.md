# Reshine 义工团网站 · 内容更新教程（零基础版）

这份教程写给**完全不会编程的同事**。所有操作都在 GitHub 网页上点鼠标完成，不需要装任何软件、不需要敲命令。

## 一、先理解这个网站怎么运作（1 分钟）

- 网站的全部内容（文字、照片）都存在 GitHub 上的 **yigong 仓库**里，它就像一个网盘
- 部署到 Vercel 之后，**你每次在 GitHub 上改动内容，网站会在 1~2 分钟内自动更新**，不需要做任何其他事
- 所以"更新网站" = "往 GitHub 仓库里传文件 / 改文字"

---

## 二、上传活动照片（最常用 ⭐）

照片不用改任何代码，只要放进对应的文件夹即可。规则只有两条：

> **规则 1：** 照片放在 `public/photos/活动英文名/` 文件夹里
> **规则 2：** 文件名就是照片说明，格式：`序号-说明文字.jpg`，例如 `01-出发前的全员合影.jpg`

仓库里已经放好了一个示例：`public/photos/riverside-cleanup-2024/`，照着做就行。

### 网页版操作步骤

1. 打开 github.com，登录后进入 **yigong** 仓库
2. 依次点进 `public` → `photos` 文件夹
3. 点右上角 **Add file → Upload files**
4. **关键一步**：在上方的路径框里，你可以输入"活动英文名/"来进入（或新建）活动文件夹。
   - 已有文件夹：直接点进那个文件夹再上传
   - 新活动：上传界面的路径栏末尾输入 `活动英文名/`（注意最后有个斜杠），GitHub 会自动创建文件夹
5. 把照片从电脑里拖进虚线框
6. 滚动到底部，点绿色的 **Commit changes** 按钮
7. 等 1~2 分钟，刷新网站，照片就出现了（包括首页粒子墙、时间轴封面、详情页大图全部自动更新）

### 照片小贴士

- **横版照片效果最好**（网站照片框是 3:2 横版）
- 单张建议压缩到 **2MB 以内**，太大会拖慢加载（手机拍的照片可以先用微信发给自己再下载，会自动压缩）
- 顺序按文件名排，所以用 `01-`、`02-` 开头控制先后
- 文件名里**不要**出现 `/ \ : * ? " < > |` 这些符号
- 第一张照片（01）会自动成为这个活动的**封面**

## 三、新增一次活动

活动的文字信息（标题、日期、介绍）在一个文件里：`src/lib/data.ts`。

1. 在 GitHub 仓库里点进 `src` → `lib` → `data.ts`
2. 点右上角的**铅笔图标 ✏️**（Edit this file）
3. 找到 `export const events` 那一段，把下面的模板**完整复制**，粘贴到 `[` 后面（其他活动的前面），然后改成你的内容：

```
  {
    slug: "tree-planting-2026",
    title: "春季植树活动",
    date: "2026-03-12",
    location: "城郊森林公园",
    summary: "一句话简介，会显示在列表里。",
    description:
      "活动的详细介绍，可以写长一点，会显示在详情页。",
    tags: ["环保"],
    participants: 40,
    hours: 4,
    cover: 701,
    photos: [],
  },
```

4. 改的时候注意：
   - `slug` 是活动的**英文代号**，只能用小写字母、数字和中横线，**它也是照片文件夹的名字**
   - `cover` 是个编号，随便填一个别的活动没用过的数字即可（如 701、801）
   - `photos: []` 留空就好，照片走第二节的文件夹方式上传
   - **逗号、引号、花括号一个都不能少**，照着模板的样子改，别删符号
5. 滚动到底部点 **Commit changes**
6. 然后按第二节的方法，建一个和 `slug` 同名的照片文件夹传照片

> 同样的方法也可以**修改**已有活动的文字：进 `data.ts` 点铅笔，直接改文字（只改引号里面的内容），提交。

## 四、修改义工团成员

也在 `src/lib/data.ts` 里，找到 `export const members` 那一段。每位成员长这样：

```
  { id: "m1", name: "林晓雨", role: "团长", department: "人力资源部", joined: "2021-03", activities: 42, motto: "微光成炬，温暖同行。", palette: ["#1fa45c", "#7edca4"] },
```

- 改名字、部门、宣言：只改**引号里面**的字
- 加新成员：复制一整行，粘贴在最后一个成员后面，把 `id` 改成没用过的（如 `m11`）
- `palette` 是这位成员卡片的两个渐变色，不懂就保持原样或复制别人的

## 五、公司 LOGO 和主题色

- **LOGO**：把 logo 图片命名为 `logo.png`，上传到 `public` 文件夹（方法同上传照片），导航栏会自动显示
- **主题色**：现在已经是公司 LOGO 的绿色。如果将来想微调，颜色都集中在 `tailwind.config.ts` 的 `colors` 那几行（leaf 是主绿色），改十六进制色号即可
- **首页大标语**："微光成炬" 在 `src/components/home/ParticleHero.tsx` 里搜索就能找到

## 六、常见问题

**Q：改完多久生效？**
A：提交（Commit）后 1~2 分钟，Vercel 自动重新发布。可以在 vercel.com 的项目页看到发布进度。

**Q：改坏了怎么办？**
A：GitHub 保留所有历史版本。进仓库点 **History**，找到出问题前的那次提交，可以查看和恢复，或者直接来找我（Claude）帮你修。

**Q：弹幕、点赞、评论的数据存在哪？**
A：默认是"演示模式"（数据只存在每个人自己的浏览器里）。想要全公司实时互通，需要配置 Supabase——这一步建议交给 IT 同事或回来找我，照着 `README.md` 的「接入 Supabase」一节做，10 分钟搞定。

**Q：能在手机上传照片吗？**
A：可以，用手机浏览器打开 github.com 操作，步骤一样。
